# CursorGood

CursorGood 是 ONE 内置的 AI 对话反馈模块，为 Cursor IDE 中的 AI Agent 提供持续交互通道。Agent 通过 MCP 协议向 ONE 发送消息，用户在 ONE 窗口中查看并回复，形成闭环反馈。

## 核心价值

Cursor 的 AI Agent 执行任务时，默认只能在 IDE 的聊天面板中等待输入。CursorGood 提供了一个**独立窗口**，让你可以：

- 随时查看 Agent 当前的进度和问题
- 通过快捷按钮一键回复常见指令
- 在多个 Agent 会话之间切换
- 不用切回 IDE 就能给 Agent 反馈

## 会话管理

- **多会话并行**：每个 Cursor Agent 自动创建独立会话，互不干扰
- **会话列表**：左侧侧栏展示所有活跃会话，标题随内容自动更新
- **归档**：超过 24 小时未更新的会话自动归档；已归档的会话收到新消息时自动恢复
- **待投递消息**：橙色图标标识有待发送消息的会话
- **取消发送**：尚未被 Agent 消费的消息可一键取消

## 聊天体验

- **Markdown 渲染**：AI 消息支持标题、列表、代码块（含语法高亮 + 一键复制）、加粗、斜体、链接等
- **图片支持**：支持 Cmd+V 粘贴图片，输入框内显示缩略图预览，点击可放大查看
- **快捷回复**：Agent 可预设 2~4 个快捷按钮，一键发送常见回复
- **历史消息**：支持向上加载更多历史消息，每次 20 条
- **消息复制**：右键菜单可复制任意消息内容
- **聊天导出**：单个会话可导出为 Markdown 文件

## MCP Server

ONE 启动后自动在本地注册 MCP 服务（端口 18880），Cursor IDE 可直接调用。

- **自动注册**：启动时写入 Cursor 的 MCP 配置，无需手动配置
- **规则安装**：自动安装 CursorGood 的 Cursor Rule 文件，确保 Agent 知道如何调用
- **断线恢复**：Agent 断连后重新连接时，之前排队的回复会自动送达

## 性能优化

- 启动时只加载会话元数据，不加载完整消息内容
- 选中会话时按需加载最新 20 条消息
- Markdown 解析结果缓存，避免重复计算
- 长期运行不再出现内存膨胀
