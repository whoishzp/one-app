# Changelog

All notable changes to ONE are documented in this file.
Format follows [Keep a Changelog](https://keepachangelog.com/).

## [1.0.1] - 2026-06-29

### Added
- **cursor/ 数据跨机器同步**：通过 pre-push hook 自动将各项目 `cursor/` 目录同步到独立远端仓库，支持 GitLab 和 GitHub
- **cursor-rules 统一管理**：重构 cursor-rules 仓库，通过 symlink 映射到 `~/.cursor/`，新增 `setup.sh` 一键安装

### Changed
- **版本号重置为 1.x**：首个公开发行版本，版本号从 1.0.1 起步
- **BOOTSTRAP 流程增强**：新增 cursor 数据同步检查步骤（.sync-remote + pre-push hook）
- **project_guide.mdc**：新增 cursor 数据同步初始化规则

## [3.10.0] - 2026-06-29

### Added
- **MCP 统一分发架构**：单 MCP Server 支持多模块工具路由，工具名用 `模块.动作` 命名空间
- **OneMind MCP 工具**：新增 `OneMind.listDocuments`、`OneMind.readDocument`、`OneMind.writeDocument` 三个 MCP 工具，AI 可直接读写 OneMind 知识库
- **OneMind AI 模块**：系统设置 > AI 页面新增「OneMind 知识库」模块，支持 Skill + Rule 的查看/导出/安装
- **同步后自动重启**：GitHub 恢复或同步拉取到新数据后弹窗提示，点击立即重启

### Changed
- **同步上传优化**：上传前先 fetch 检测远端是否有新提交，有则先 pull 合并再 push；pull 到新数据后提示重启
- **MCP Server 版本升级**：serverInfo 名称改为 ONE-MCP v2.0

### Fixed
- **OneMind 文档数据丢失**：切换文档时 debounced save 被取消导致数据丢失，新增 deinit 刷盘机制
- **OMIndex 编码格式**：修复 UUID 字典序列化为交替数组导致 Git 合并损坏的问题，改用 string-keyed JSON 格式（向后兼容旧格式）
- **GitHub Sync push rejected**：远端领先时 push 被拒绝，新增 fetch + rebase 回退机制

## [3.9.28] - 2026-06-25

### Added
- **默认背景图**：应用内置默认背景图，首次安装即有背景效果；用户可自定义替换，清除后恢复默认
- **聊天头部选中复制**：CursorGood 聊天头部的 session 标题和完整 ID 支持鼠标选中复制

### Changed
- **默认透明度调整**：背景图默认透明度 30% → 55%，面板默认透明度 100% → 35%
- **侧栏 session 标题**：CursorGood 会话列表标题从单行截断改为最多两行展示
- **清除背景改为恢复默认**：主题设置中「清除」按钮改为「恢复默认」，点击后回退到内置默认背景

## [3.9.12] - 2026-06-23

### Added
- **软件更新设置**：「关于 ONE」页面新增「软件更新」卡片，支持开关自动检测（默认每小时）和手动检测更新按钮

### Fixed
- **自动更新重启卡死**：修复点击「立即重启」后新旧进程并存、旧进程卡住的问题；改用 shell 子进程延迟重启 + `exit(0)` 强退

## [3.9.9] - 2026-06-23

## [3.9.7] - 2026-06-23

### Fixed
- **OneMind 授权弹窗**：修复每次打开 OneMind 弹出「想访问文稿文件夹」的 TCC 授权弹窗。根因是 SPM 生成的 `Bundle.module` 在 `.app/` 根目录找不到 `ONE_ONE.bundle` 后 fallback 到硬编码的构建目录（位于 ~/Documents）。别人电脑上该路径不存在会直接崩溃
- **WKWebView file:// 依赖**：编辑器资源和用户图片改用 `one-res://` 自定义协议加载，完全消除 WebView 对 `file://` 的依赖

### Added
- **取消发送回填输入框**：CursorGood 点击「取消发送」后，消息内容自动回填到输入框

### Changed
- **build.sh 打包校验**：构建完成后自动校验 `ONE_ONE.bundle` 是否在 SPM 期望的位置（`.app/` 根目录），不存在则中止构建

## [3.9.6] - 2026-06-23

### Fixed
- **CursorGood 性能优化**：Session 消息懒加载，启动只加载元数据，选中 session 时加载最新 20 条消息，解决长期运行后内存膨胀（1.5GB+）和主线程卡死问题
- **Markdown 解析缓存**：AI 消息的 Markdown 解析结果使用 NSCache 缓存，避免每次 SwiftUI 布局重复计算
- **ContentBlock ID 稳定化**：改用 `messageId-index` 格式替代 hashValue，消除 SwiftUI diff 碰撞风险
- **修复加载更多滚动**：点击「加载更多历史消息」后保持当前阅读位置，不再跳到底部
- **修复初次渲染闪跳**：首次打开聊天时直接定位到最新消息，无可见滚动动画

### Added
- **Pending 消息取消**：用户发送的待投递消息新增「取消发送」按钮，可撤回未被 AI 消费的消息
- **Pending 索引独立存储**：新增 `pending-index.json`，启动时不再全量扫描 session 文件
- **历史消息分页加载**：聊天界面顶部显示「加载更多历史消息」按钮，每次加载 20 条
- **启动兼容校正**：首次升级时自动从旧 session 文件生成 pending 索引；损坏的 session 文件自动清理并打日志；缺失字段自动填充默认值
